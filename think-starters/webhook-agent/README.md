# Think Webhook Agent Starter

A [Think](https://www.npmjs.com/package/@cloudflare/think) agent that durably
processes inbound webhook events, with a small React chat UI to watch it work.

The webhook handler accepts an external POST, returns immediately, and hands the
event to a durable agent with `submitMessages({ idempotencyKey })`. Because
admission is durable and idempotent, a retried or duplicated webhook never starts
a second turn — the core reliability property for event-driven agents.

## Develop

```sh
npm install
npm run dev
```

Open the printed URL to watch the agent. Then send it an event:

```sh
curl -X POST http://localhost:5173/webhook \
  -H "content-type: application/json" \
  -d '{ "id": "evt_123", "type": "payment.disputed", "amount": 4200 }'
```

The response returns immediately with a `submissionId` and `accepted: true`. POST
the same `id` again and you get `accepted: false` — the duplicate is ignored. The
processed event appears in the chat UI as the agent reasons about it.

## What's here

- `agents/inbox/agent.ts` — the agent: a system prompt for triaging events plus a
  mocked `notifyTeam` tool.
- `src/server.ts` — a custom Worker entry that runs before the Think router. It
  handles `POST /webhook` and durably submits the event; everything else falls
  through to Think.
- `src/client.tsx` — a React chat client connected to the same `default` inbox
  instance the webhook feeds.
- `wrangler.jsonc` / `think.d.ts` — generated by the Think framework. Re-run
  `npm run types` after changing agents or bindings.

## Deploy

```sh
npm run deploy
```
