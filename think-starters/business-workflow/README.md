# Think Business Workflow Starter

A [Think](https://www.npmjs.com/package/@cloudflare/think) back-office operations
agent on Cloudflare Workers, with human-in-the-loop approval gates and a small
React chat UI.

The agent runs a real business process — look up an account, then take a
sensitive action — but pauses for human sign-off on the risky steps. `issueRefund`
asks for approval above $100; `closeAccount` always does. While a turn waits on a
human, Think parks it instead of failing it, so a deploy or timeout during the
wait does not lose the pending decision.

## Develop

```sh
npm install
npm run dev
```

Open the printed URL and try:

> Refund $250 to ACC-4821 for a billing error

The agent looks up the account, then surfaces an **Approve / Reject** card before
issuing the refund. Approving resumes the turn and runs the tool; rejecting tells
the model the action was declined so it can respond.

## What's here

- `agents/operations/agent.ts` — the agent: `lookupAccount`, plus `issueRefund`
  and `closeAccount` guarded by `needsApproval`, and a scheduled `dailyDigest`.
- `src/client.tsx` — a React chat client that renders approval cards and resolves
  them with `addToolApprovalResponse()`.
- `wrangler.jsonc` / `think.d.ts` — generated by the Think framework. Re-run
  `npm run types` after changing agents or bindings.

## Deploy
