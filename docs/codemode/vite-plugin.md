# Vite Plugin

The `@cloudflare/codemode/vite` plugin handles connector discovery and auto-export so users don't need manual wrangler configuration.

**Why this exists:** connectors extend `WorkerEntrypoint` and must be exported from the worker entry module to be reachable over Workers RPC, and the `CodemodeRuntime` facet class must be exported for facet spawning. That's plumbing nobody should hand-maintain — the plugin derives it from a file convention (`*.codemode.ts`) instead.

## Setup

```ts
// vite.config.ts
import codemode from "@cloudflare/codemode/vite";
import agents from "agents/vite";
import { cloudflare } from "@cloudflare/vite-plugin";

export default {
  plugins: [agents(), codemode(), cloudflare()]
};
```

## What it does

### 1. Discovers `*.codemode.ts` files

The plugin finds all files matching `*.codemode.{ts,js,tsx,jsx}` in the project's `src/` directory.

### 2. Handles `with { type: "connectors" }` imports

When your code imports with the `type: "connectors"` attribute:

```ts
import { GithubConnector } from "./github.codemode" with { type: "connectors" };
```

The plugin resolves the import through a virtual module that re-exports the connector classes. This works for both single files and directories:

```ts
// Single file
import { GithubConnector } from "./github.codemode" with { type: "connectors" };

// Directory — all *.codemode.ts files in it
import * as connectors from "./connectors" with { type: "connectors" };
```

### 3. Auto-exports from the worker entry

The plugin appends re-exports to the worker entry module (`src/server.ts`, `src/index.ts`, or `src/worker.ts`):

```ts
// Auto-generated by @cloudflare/codemode/vite
export { CodemodeRuntime } from "@cloudflare/codemode";
export * from "/abs/path/to/github.codemode.ts";
export * from "/abs/path/to/repoapi.codemode.ts";
```

This makes connector classes and the session class available as `ctx.exports.*` in the Workers runtime, which is required for facet spawning.

### 4. Watch mode

In development, the plugin watches `*.codemode.ts` files for changes and re-exports are updated on HMR.

## File convention

Connector files use the `*.codemode.ts` extension:

```
src/
  github.codemode.ts        → GithubConnector
  stripe.codemode.ts        → StripeConnector
  linear.codemode.ts        → LinearConnector
  server.ts                 → agent + createCodemodeRuntime
  vite.config.ts            → plugins: [codemode()]
```

## How the worker entry looks after the plugin

Before (what the user writes):

```ts
// src/server.ts
import { GithubConnector } from "./github.codemode" with { type: "connectors" };

export class Chat extends AIChatAgent<Env> { ... }

export default { async fetch(request, env, ctx) { ... } };
```

After (what the plugin emits):

```ts
// src/server.ts (transformed)
import { GithubConnector } from "__codemode_connectors__:...";

export class Chat extends AIChatAgent<Env> { ... }

export default { async fetch(request, env, ctx) { ... } };

// Auto-exported by @cloudflare/codemode/vite
export { CodemodeRuntime } from "@cloudflare/codemode";
export * from "/abs/path/to/github.codemode.ts";
export * from "/abs/path/to/repoapi.codemode.ts";
```

The Workers runtime sees `GithubConnector`, `RepoApiConnector`, and `CodemodeRuntime` as top-level exports → they're available as `ctx.exports.*`.

## TypeScript

For import attributes to work, the example's `tsconfig.json` needs:

```json
{
  "extends": "agents/tsconfig",
  "compilerOptions": {
    "module": "esnext"
  }
}
```

The `esnext` module setting enables `import ... with { type: "..." }` syntax.
